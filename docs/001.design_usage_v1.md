# Agent Pair Programming

Standalone Playwright-based ChatGPT automation runtime and CLI.

Module-level behavior docs live in source docstrings under `src/navigator/*`.
The CLI module boundary lives in `src/modules/pp/module.ts`.

## Auth and Session Modes

- `--cdp-url` attaches to an existing browser instance and leaves lifecycle ownership external.
- `--user-data-dir` launches a persistent Chromium context with local profile state.
- `--auth-file` launches an ephemeral Chromium context seeded from Playwright `storageState`.
- `--auth-file` cannot be combined with `--cdp-url` or `--user-data-dir`.
- Default state root: `${XDG_STATE_HOME:-~/.local/state}/pp`
- Default profile dir: `${XDG_STATE_HOME:-~/.local/state}/pp/profiles/chatgpt-profile`
- Default auth dir: `${XDG_STATE_HOME:-~/.local/state}/pp/auth`
- Reusable session metadata: `${XDG_STATE_HOME:-~/.local/state}/pp/runtime`

## Project Persistence Policy

- Project persistence is env-only.
- `PP_CHATGPT_PROJECT` is the only default project source.
- `--project` overrides the env value for the current command.
- Profile persistence is env-only.
- `PP_PROFILE` maps to `${XDG_STATE_HOME:-~/.local/state}/pp/profiles/<name>` when `--user-data-dir`/`--profile` are omitted.
- `--profile` overrides the env value for the current command.
- There is no profile mutation command.
- Project values accept `g-p-*` IDs or ChatGPT project/conversation URLs.

## Command Groups

- Operational loop: `pp send`, `pp wait`, `pp get-response`, `pp history`
- Composition loop: `pp compose`, `pp brief`
- Attachment/download loop: `pp attach`, `pp paste`, `pp download`
- Session/model controls: `pp set-model`, `pp new`, `pp refresh`, `pp isolate`
- Auth export utility: `auth-listen`

## Runtime Parity Notes

- `pp wait`, `pp get-response`, `pp history`, `pp download`, and `pp paste` default to non-navigating session attach mode.
- An explicit `--project` on non-navigating commands performs an intentional redirect unless `--no-navigate` is passed.
- `pp send` preserves dedupe and conversation-cap block semantics and emits parseable `snake_case` compatibility fields.
- `pp brief` now packages `--entries` selections into a max-compressed `tar.gz` attachment plus manifest by default; `--inline-entries` restores inline file-body pasting.
- `pp new` defaults the model mode to `thinking` when omitted.
- Tab targeting prefers active project/conversation ChatGPT tabs and uses deterministic newest-tab tie-breaking.

## Test Matrix

- `tests/cli.test.mjs` covers `pp` command parsing and validation.
- `tests/navigator/*.test.mjs` covers parser/runtime helpers, session navigation policy, tab selection, env project policy, and limits behavior.
- `tests/playwright/navigator/*.spec.ts` covers browser behavior for compose/send/response/download/attachment/paste flows.
- `tests/playwright/navigator/chatgpt_live_roundtrip.spec.ts` is opt-in live ChatGPT verification via env flags.
